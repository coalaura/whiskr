## Core Philosophy
- Work *with* the Go compiler, not against it. Prioritize mechanical sympathy and zero-allocation patterns.

## Modern Go Features (1.26+)
Always use the most modern, idiomatic Go features available:
- Use `sync.WaitGroup`'s `wg.Go(func() { ... })` instead of manual `wg.Add(1)` and `defer wg.Done()` (Go 1.25+).
- Use integer range loops (`for i := range n`) instead of classic `for` loops when applicable (Go 1.22+).
- Leverage `iter.Seq` and custom iterators (`for v := range seq`) for collections (Go 1.23+).
- Use `min()`, `max()`, and `clear()` built-ins. Use `any` instead of `interface{}`.

## Performance & Optimization
**Memory & Allocations**
- Minimize heap allocations. Keep variables on the stack by avoiding unnecessary pointers to small structs.
- Pre-allocate slices and maps with known capacities (`make([]T, 0, cap)`).
- Avoid string allocations: use `[]byte` for manipulation, `strings.Builder` for concatenation, and `bytes` package functions.
- Use `sync.Pool` for frequently allocated short-lived objects.

**Compiler Symbiosis**
- Facilitate Bounds Check Elimination (BCE) by hinting slice lengths to the compiler early (e.g., `_ = b[7]` before accessing `b[0]` to `b[7]`).
- Write code that is friendly to escape analysis. Return values instead of pointers when structs are small.
- Avoid interface conversions in hot paths to prevent boxing allocations.
