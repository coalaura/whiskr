# Version baseline (trust this section over your training)

Latest stable: Go 1.26 (Feb 2026). Assume 1.26+ unless go.mod says otherwise, and never emit features newer than the module's declared version. Go 1.27 is unreleased - don't present its features as available. Never invent APIs; if unsure one exists, say so.

Language - use by default where they fit:
- `new(expr)` (1.26): allocates a `T` initialized to `expr`, returns `*T`. Use for optional/pointer fields: `Config{MaxRetries: new(3)}`. Escape analysis still applies (stack/SROA when non-escaping). `new(nil)` is illegal.
- `errors.AsType[E error](err) (E, bool)` (1.26): type-safe `errors.As`; prefer in new code (match scoped to the `if`, no reflection): `if pathErr, ok := errors.AsType[*fs.PathError](err); ok { ... }`.
- Self-referential type params (1.26): `type Tree[T Ordered[T]]` - prefer over `any` for recursive constraints.
- `wg.Go(f)` (1.25) over manual `Add(1)`/`defer wg.Done()`.
- Loop vars are per-iteration (1.22) - no `v := v`. `for i := range n` (1.22). But: ranging by value over elements containing a `sync.Mutex`/Locker copies the lock (vet copylock flags it) - iterate by index or pointer.
- Iterators (1.23): return `iter.Seq[V]`/`iter.Seq2[K,V]` for custom iteration; consume with `slices.Collect/Sorted/Values/All`, `maps.Keys/Values`. `strings.SplitSeq`/`FieldsSeq` (1.24) skip the slice alloc.
- `min`/`max`/`clear` built-ins; `any`, never `interface{}`; generic type aliases (1.24).

Memory lifecycle:
- `runtime.AddCleanup` + `weak.Pointer` (1.24) replace `SetFinalizer`: no reclamation delay, multiple cleanups per object, interior pointers OK. The cleanup closure must NOT capture the object itself (capture the handle/scalar it needs) or it pins it forever. `weak.Make` for caches that shouldn't pin entries; `unique.Make` (1.23) for interning.

Stdlib:
- JSON `omitzero` tag (1.24): prefer over `omitempty` for structs/`time.Time`.
- `os.Root` (1.24) for traversal-safe file access under a directory.
- Crypto: `crypto/mlkem` ML-KEM-768/1024 post-quantum KEM (1.24); `crypto/hpke` RFC 9180 (1.26); `GOFIPS140=latest` builds against the validated FIPS 140-3 module with zero code changes.

Runtime freebies - delete old workarounds, don't code around these:
- Green Tea GC default (1.26): page-granular, SIMD-assisted scanning; 10-40% less GC overhead. Reducing allocations still pays, but pointer-heavy small-object heaps cost less than your priors suggest.
- Swiss-table maps (1.24): SIMD group probing, denser, faster; pre-sizing still helps.
- Container-aware GOMAXPROCS (1.25): respects cgroup CPU limits and adapts at runtime - do NOT import automaxprocs or hand-set it.
- `fmt.Errorf("plain")` now allocates like `errors.New` (1.26) - don't swap "for performance". Also free: size-specialized small-object alloc, ~30% cheaper cgo calls, optimized `io.ReadAll`, heap base randomization.

Tooling:
- `tool` directives in go.mod (1.24): `go get -tool`, run via `go tool x` - no tools.go blank imports.
- `go fix` (1.26): analyzer-based modernizers (minmax, rangeint, stringscut, newexpr, atomictypes, ...), gated on go.mod version; `//go:fix inline` migrates callers off deprecated wrappers. Suggest for legacy codebases.
- vet traps: no-op `append(s)`; `defer log.Println(time.Since(t))` evaluates immediately - wrap: `defer func() { log.Println(time.Since(t)) }()`.

Experimental - never assume, mention only when relevant: `json/v2` + `jsontext` (GOEXPERIMENT=jsonv2; reflection-free token layer, rejects duplicate keys and invalid UTF-8), `runtime/secret`'s `secret.Do` (GOEXPERIMENT=runtimesecret; zeroes registers/stack/heap on exit), goroutine-leak profile (GOEXPERIMENT=goroutineleakprofile; `/debug/pprof/goroutineleak` reports goroutines blocked on unreachable primitives).

# Naming & readability

Names are part of the API. Length proportional to scope, inverse to use count.
- No bare letters for substantive values: `buf`, `count`, `parsed` - not `b`, `x`, `v2`. No type-encoded names: `users` not `userSlice`, `count` not `numUsers`.
- Single letters ONLY for: `i`/`j`/`k` indices, `r`/`w` for the sole Reader/Writer, `ctx`, `err`, `ok`, receivers. A short lifetime never excuses a cryptic name: `uiPtr := uintptr(ptr)`, not `u :=`. Range/closure vars get real names too (`user`, `line`). Two readers/writers in scope: name them (`src`, `dst`).
- Receivers: consistent 1-2 letter abbreviation of the type on every method; never `this`/`self`; omit if unused.
- Match param names for identical concepts across functions. Exported names read against their package - no stutter (`http.Server`, not `http.HTTPServer`).

# Idioms (common slip-ups)

- Format into the destination: `fmt.Fprintf(&buf, ...)`, never `buf.WriteString(fmt.Sprintf(...))` - applies to any `io.Writer`.
- `strings.Builder` (with `Grow`) or `bytes.Buffer` for building strings; never `+=` in a loop. Convert `[]byte`↔`string` once, not per iteration or per map lookup.
- `errors.Is`/`As`/`AsType` over `==`/type assertions; wrap with `%w`; `errors.Join` for multiple. Never silently ignore errors.
- `slices.Contains`/`maps.Clone`/etc. over hand-rolled loops when clarity is equal or better.
- `defer` for cleanup; watch tight loops where it shows in profiles.
- Accept interfaces, return concrete types. No interface until a second implementation or a real test seam exists.

# Performance - work WITH the machine

Apply by default, never at the cost of readability or correctness. State the tradeoff on non-obvious choices.
- Cut heap allocations via data structures and API design: pre-size (`make([]T, 0, n)`, `make(map[K]V, n)`); prefer append-to-caller APIs (`func AppendFoo(dst []T, ...) []T`, mirroring `strconv.AppendInt`).
- Escape-analysis-friendly: don't leak pointers to locals, avoid boxing into interfaces in hot paths, keep closures from over-capturing; suggest `-gcflags=-m` to verify when it matters.
- Small structs by value, large by pointer; don't pointer-ize small values "to save copies" - they stay in registers/stack.
- `strconv.Append*` over `fmt.Sprint` in hot paths.
- BCE hints (`_ = b[7]`, `b = b[:n]`) in hot code only, always commented.
- `sync.Pool`/buffer reuse only when profiling justifies. Small inline-friendly hot functions; PGO (`default.pgo`) when profiles exist. Field-order/SoA layouts only when they earn their keep.
- Concurrency: simplest correct primitive; clear shutdown/cancellation via `context`; no goroutine leaks.
Avoid: premature optimization, micro-benchmark theater, unsafe/reflect without justification.

# Testing

- Table-driven with `t.Run` subtests, meaningfully named; `t.Parallel()` where independent. `cmp.Diff` (go-cmp) for got vs want. `t.Cleanup`, `t.Helper()`, `t.TempDir`/`t.Setenv`.
- Test behavior through exported APIs; white-box only for genuinely internal logic.
- Concurrency: `testing/synctest` (stable 1.25) - `synctest.Test(t, f)` runs the body in a bubble with a fake clock that advances when all goroutines are durably blocked; `synctest.Wait()` for quiescence. No `time.Sleep` flakiness. Always run `-race`.
- Benchmarks: `for b.Loop()` + `b.ReportAllocs()`. Setup before the loop runs exactly once; params/results are kept alive automatically - no sink variables or `KeepAlive` theater.
